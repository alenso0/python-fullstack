All three does the same task

Takes in HTTP request and generate responses

- FLASK : Can be built it in your own way - minimal - flexible - comes with routing - request response (Small to medium apps) - more flexibility

- Django : Do it the right way - Factory with some pre-defined rules - Database layer , Authentication, admin panel etc (Large application)

- FastAPI : API focused excellent performance - used for API, microServices

- Function Excecutes and return values captured - HTTP response is generated - given to brownser and it displays it 

- Static no rendereing 
- template rendered by flask in runtime
